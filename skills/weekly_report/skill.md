# Weekly Report Skill（项目周报生成技能）

## 目标
把“任务清单（Excel/CSV）”转成一份符合公司口径的项目周报（Markdown）。

## 输入
- 用户一句话需求（自然语言）
- 汇总 KPIs（JSON）
- 任务表（Markdown 表格，截取前若干行）

## 执行 SOP（必须遵循）
1. 先给出【总体概览】（总任务数、完成/进行中/阻塞/未开始、逾期数）。
2. 再按【项目维度】输出进度（按 project 汇总 done/doing/blocked/todo）。
3. 输出【Top 风险/阻塞】（从 top_risks 里挑 3～5 条，写清：影响、原因、建议、Owner）。
4. 输出【负责人维度】（按 owner 汇总，标出阻塞/逾期的负责人）。
5. 输出【下周计划/行动项】（必须可执行：动作 + Owner + 截止日期；如果缺日期写“待补充”）。
6. 输出【需要领导决策/资源支持】（如果数据里体现阻塞/风险，必须写 1～3 条需求）。

## 口径与约束
- 只能基于输入数据输出事实；不要编造指标、里程碑、项目。
- 数据缺失就写“待补充”，不要猜。
- 输出必须严格使用 report_template.md 的结构和标题。
- 用简洁、管理层友好的表达：结论先行，风险明确，行动闭环。
